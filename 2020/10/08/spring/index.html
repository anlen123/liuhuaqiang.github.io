package com.lhq.pojo;

import org.springframework.beans.factory.annotation.Autowired;

public class Person {
    //定义了这个required为false表示,这个属性可以为空
    @Autowired(required = false)
    private Cat cat;
    @Autowired
    private Dog dog;
    private String name;

    @Override
    public String toString() {
        return "Person{" +
                "cat=" + cat +
                ", dog=" + dog +
                ", name='" + name + '\'' +
                '}';
    }

    public Cat getCat() {
        return cat;
    }


    public Dog getDog() {
        return dog;
    }


    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }
}

小结: @Resource 和 @Autowired 区别

  • 都是用来自动装配的,可以放在字段名上
  • @ AutoWired 通过ByName 的方式实现(必须要求这个字段存在)
  • @Resource 默认通过ByName 方式实现,如果找不到名字,则通过ByTyope实现,如果两个都找不到的情况下,报错
  • 执行顺序不同

为什么要学习代理模式? 因为这是SpringAOP的底层[SpringAOP和SpringMVC]

代理模式的分类:

  • 静态代理
  • 动态代理

静态代理

角色分析:

  • 抽象角色: 一般用接口或者抽象类来解决
  • 真是角色: 被代理的角色
  • 代理角色: 代理真是角色后,我们一般会进行一些附属操作
  • 客户: 访问代理对象的人

房东:

public class Host implements Rent{

package com.lhq.demo01;

public class Proxy implements Rent {
    private Host host;

    public Proxy() {
    }

    public Proxy(Host host) {

        host.rent();

代理模式的好处

  • 可以是真是角色的操作更加纯粹
  • 公共的就交给代理角色,实现了业务的分工
  • 公共业务发生修改时,可以方便集中管理

缺点:

  • 一个真是角色会产生一个代理角色,代码量翻倍

  • 动态代理和静态代理角色一样
  • 动态代理的代理类是动态代理的,不是我们自己直接 写好的
  • 动态代理,分为两大类,,基于接口的动态代理,基于类的动态代理
    • 基于接口-JDK动态代理
    • 基于类: cglib
    • java字节码实现: javasist

代理者

public class ProxyInvocation implements InvocationHandler {
    private Object Object;

    public void setRent(Object Object) {
        this.Object = Object;
    }

    public Object getProxy() {
        return Proxy.newProxyInstance(this.getClass().getClassLoader(), Object.getClass().getInterfaces(), this);
    }

    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        log(method.getName());
        Object invoke = method.invoke(Object, args);
        return invoke;
    }

    public void log(String msg) {
        System.out.println("msg: " + msg);
    }
}

package com.lhq.Diy;

import org.aspectj.lang.ProceedingJoinPoint;
import org.aspectj.lang.annotation.After;
import org.aspectj.lang.annotation.Around;
import org.aspectj.lang.annotation.Aspect;
import org.aspectj.lang.annotation.Before;

import java.net.SocketTimeoutException;

@Aspect //这个类是一个切面
public class AnnotationPointCut {
    @Before("execution(* com.lhq.service.UserServiceImpl.*(..))")
    public void before() {
        System.out.println("==========before===========");
    }

    @After("execution(* com.lhq.service.UserServiceImpl.*(..))")
    public void after() {
        System.out.println("==========after============");
    }

    @Around("execution(* com.lhq.service.UserServiceImpl.*(..))")
    public void around(ProceedingJoinPoint joinPoint) throws Throwable {
        System.out.println("环绕前~~");
        Object proceed = joinPoint.proceed();
        System.out.println("环绕后~~");
    }
}

环绕后~~
==========after============
